# Telegram как вход интервьюера

Текстовые вопросы из Telegram попадают в `data/transcript.md` как `[Интервьюер]:` — так же, как после STT с BlackHole.

## Настройка

1. В Telegram: [@BotFather](https://t.me/BotFather) → `/newbot` → скопируй **токен**.
2. Узнай свой **chat id**: [@userinfobot](https://t.me/userinfobot) → `/start`.
3. В `.env`:

```bash
TELEGRAM_INPUT_ENABLED=1
TELEGRAM_BOT_TOKEN=123456:ABC...
TELEGRAM_CHAT_IDS=123456789
# TELEGRAM_CHAT_ID=123456789   # один id — то же самое
```

4. Напиши боту любое сообщение (чтобы чат «открылся»).
5. При запуске `copilot` sidecar держит long-poll Telegram до **CP → Выход** (не гасится на **Закончить интервью**).

## Поведение

- Сообщения только из чатов из `TELEGRAM_CHAT_IDS` (безопасность).
- **Текст** и **голосовые** (кружки/войсы) в чате с ботом → STT → `[Интервьюер]:`.
- В терминале — блок **Интервьюер** (как при STT).
- **⌘↩** — ответ (DeepSeek / Cursor); подряд идущие реплики Telegram + STT склеиваются в один вопрос.
- **⌘⌃⇧4** (скриншот) **не блокирует** ⌘↩ — ответ и скрины идут параллельно (очередь скринов + отдельный поток ответа).
- **⌘G** — очистить транскрипт.

Можно совмещать с аудио (`AUDIO_ENABLE_INTERVIEWER=1`) или отключить звук интервьюера и слать вопросы только из Telegram.

## Голосовой звонок в Telegram ≠ бот

Это частая путаница: **звонок человеку** и **чат с ботом** — разные каналы.

| | Чат с **ботом** (текст) | **Звонок** в Telegram (VoIP) |
|--|-------------------------|------------------------------|
| Куда идёт | Сообщения в личку боту | Звонок обычному контакту |
| Copilot | `TELEGRAM_*` в `.env` | **Не бот** — только аудио + STT |
| Что нужно | Токен, chat id, `TELEGRAM_INPUT_ENABLED=1` | BlackHole + «Начать прослушивание» |

**Бот не слышит звонки.** Bot API принимает только текст. Исходящие/входящие звонки (как в истории «Outgoing Call») copilot через бота не получит.

### Если интервьюер говорит в звонке Telegram

Настраивается как Zoom/Meet — см. [audio-setup.md](audio-setup.md):

1. **Не** выбирай BlackHole в **Системные настройки → Звук → Вход** (микрофон системы). Copilot пишет с BlackHole то, что **приложение отдаёт на выход**.

2. **Telegram → Настройки → Звук и видео:**
   - **Микрофон** — твой реальный (Brio и т.п.), чтобы тебя слышали в звонке.
   - **Динамик / устройство воспроизведения** — `BlackHole 2ch` **или** Multi-Output (BlackHole + наушники). Именно **выход** голоса собеседника должен попадать в BlackHole.

3. В `.env` sidecar:
   ```bash
   AUDIO_INPUT_INTERVIEWER=BlackHole
   AUDIO_ENABLE_INTERVIEWER=1
   AUDIO_INPUT_SELF=Brio          # твой микрофон
   ```

4. **CP → Начать интервью** → **Начать прослушивание** → звонок в Telegram.

5. В терминале должны появляться блоки **Интервьюер** после пауз в речи (Whisper).

**Multi-Output:** если поставить только BlackHole без наушников в Multi-Output — в звонке будет тишина в ушах. Собери Multi-Output в Audio MIDI Setup (BlackHole + наушники), выбери его динамиком в Telegram.

### Два входа сразу

- **Текст боту** → `[Интервьюер]:` через Telegram API.
- **Голос в звонке** → тот же тег через STT с BlackHole.

Оба могут работать параллельно; на **⌘↩** склеивается последний блок подряд идущих `[Интервьюер]` (и текст, и STT).

## Рекомендуемый `.env` для интервью + скрины

```env
ANSWER_PROVIDER=deepseek          # ⌘↩ — быстрый текст, не блокируется скринами
SCREENSHOT_ANSWER_PROVIDER=cursor # ⌘⌃⇧4 — vision через Cursor SDK
TELEGRAM_INPUT_ENABLED=1
```

`ANSWER_PROVIDER=cursor` для ⌘↩ и скрин одновременно делят один subprocess SDK — для параллельной работы лучше схема выше.

## Команды бота

- `/start`, `/help` — краткая подсказка.

Зависимостей сверх stdlib нет (HTTP через `urllib`).
