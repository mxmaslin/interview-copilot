# Copilot (копилот)

Личный помощник для прохождения технических и прочих интервью. Не SaaS: только автор, бесплатно для себя (в отличие от InterviewCoder / Ultracode — без подписок на сторонние копилоты).

**Главная идея:** всё в привычном интерфейсе **Cursor** — отдельный **спец-агент** в IDE; отдельное оверлей-приложение не нужно. Экран редко просят шарить — Cursor на экране допустим.

---

## Как это работает (два слоя)

### 1. Спец-агент в Cursor (основной UX)

- Агент с правилами (`.cursor/rules`, `AGENTS.md`): Python backend, RU + EN термины, краткие ответы («определение → пример → оговорки»).
- Сессию стартует **sidecar (CP → Начать интервью)**, не фразы в чате.
- Ответы и транскрипт — в run агента Cursor и `data/transcript.md`.

### 2. Тонкий audio-sidecar на macOS (обязателен)

Cursor **не умеет** сам слушать звук из Zoom / Meet. Небольшой фоновый процесс на Mac:

- захватывает аудио созвона (системный вывод + при необходимости микрофон);
- прогоняет **STT** (облако или локальный Whisper — офлайн не обязателен);
- дописывает реплики в `data/transcript.md` (`[Интервьюер]` / `[Я]`) и при ⌘↩ — **ответ в терминал** + `data/last-answer.md`.

Без sidecar «агент слушает» технически невозможен — это только обвязка вокруг агента (~10–20% объёма проекта).

---

## Сценарий на интервью

1. Открыть репозиторий в Cursor.
2. Терминал: `copilot` → menubar **CP**.
3. **CP → Начать интервью**.
4. Реплики: STT и/или **Добавить реплику интервьюера**.
5. **`⌘↩`** — ответ в терминале (и в `data/last-answer.md`). **`⌘G`** — очистить транскрипт.
6. **CP → Закончить интервью** → **Выход**.

## Интеграция с Cursor

- **Основной UX** — терминал, где запущен `copilot`; Cursor не перехватывает фокус.
- **Генерация** — DeepSeek/OpenAI API (или SDK при `ANSWER_PROVIDER=cursor`).
- **Agents** — опционально вручную (New Agent) + `CURSOR_AGENT_MIRROR=1`.
- Hotkey **`Cmd + Enter`** — sidecar.

Системный промпт sidecar’а при отправке: последний блок `[Интервьюер]: …`, режим interview, Python backend, RU + EN.

---

## Аудио на macOS (технические заметки)

- Захват звука созвона: виртуальное аудиоустройство (например BlackHole) + маршрутизация вывода Zoom / Telemost / Meet.
- Разделение «кто говорил»: по возможности два канала (системный звук vs микрофон); иначе — эвристики или ручная пометка в sidecar.
- Задержка end-to-end (STT + агент): ориентир 3–15 с — приемлемо для большинства теоретических вопросов.

## MVP (по этапам)

1. ~~**Спец-агент**~~ — правила + ручная вставка вопроса → ответ в чате.
2. ~~**Sidecar: транскрипт**~~ — файл `data/transcript.md`, менubar.
3. ~~**Hotkey `Cmd + Enter`**~~ — ответ через Cursor SDK.
4. **STT + захват звука** — BlackHole + локальный faster-whisper (по умолчанию) или OpenAI API; меню «Начать прослушивание» (`docs/audio-setup.md`).
5. Дальше: diarization `[Интервьюер]`/`[Я]`, авто-push реплик в чат Agents без ⌘↩.

---

## Вне scope (пока)

- Оверлей, отдельное desktop-приложение с UI.
- Мультипользовательский продукт, монетизация.
- «Невидимость» при шаринге экрана (шаринг запрашивают редко — не приоритет).
